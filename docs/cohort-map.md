## How to read this

Each non-`base` cohort becomes **one shared virtualenv** under `--build-dir/venvs/<key>/` when you build with `--manage-venvs`. Families in `base` (no `requirements.txt`) share the base venv. So a full build creates on the order of ~216 venvs rather than one per family ‚Äî and you can shrink that further by not building the long tail of singletons.

Full machine-readable data (every cohort ‚Üí its requirements + complete family list) is produced alongside this file as `cohorts.json` in the `--build-dir`.

Regenerate with:

```sh
python3 gflib_build.py --cohorts-report \
  --google-fonts <clone> --archive <repo_archive> --build-dir <out> --jobs 16
```
